# Fix for cti0111_1.py IndexError

## Root Cause
The `IndexError: single positional indexer is out-of-bounds` at line 161 (`last_row_template = target_df.iloc[-1].copy()`) occurred when the `target_df` (loaded from `public/cti_data.csv`) was empty or did not contain any rows.

This happened because:
1. The script reads the target CSV file with `pd.read_csv(TARGET_CSV, header=2, dtype=str)`
2. When the CSV file only contains header lines (no data rows), `target_df` becomes an empty DataFrame
3. When the script tries to access `target_df.iloc[-1]` to get the last row as a template for new rows, it fails because there are no rows in the DataFrame

## Changes Made
Modified `scripts/convert_cti0111_1.py` to handle the case where `target_df` is empty:

**Location:** Lines 157-180 in the `if new_yms:` block

**Change:** Added a check to see if `target_df` has any rows before trying to access the last row:

```python
if len(target_df) > 0:
    last_row_template = target_df.iloc[-1].copy()
else:
    # If target_df is empty, create a template with empty values
    last_row_template = pd.Series({col: '' for col in target_cols})
```

This ensures that:
1. When `target_df` has data, it uses the last row as a template (original behavior)
2. When `target_df` is empty, it creates a new template with empty values for all columns
