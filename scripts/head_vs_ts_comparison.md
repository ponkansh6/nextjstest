# Git HEAD (商用) vs TS生成CSV 差分検証レポート

## 検証日

2026-06-23

## 概要

Python→TypeScript コンバーター移行後、TSが生成したCSVと現在商用で使われているgit HEAD上のCSVの差分を検証。
値の同一性、構造の変化、データ範囲の変化を確認。

---

## 1. contractual_earnings.csv

| 項目     | HEAD (git)   | WORK (TS生成) | 差分       |
| -------- | ------------ | ------------- | ---------- |
| 行数     | 163          | 168           | +5         |
| 列数     | 21           | 21            | 0          |
| サイズ   | 12,142 bytes | 13,008 bytes  | +866 bytes |
| 日付範囲 | 1952-2024    | 1952-2024     | 同範囲     |

**構造変化:**

- HEAD: 1行ヘッダー + 2行サブヘッダー + データ
- WORK: 6行メタデータ + 1行ヘッダー + 2行サブヘッダー + データ

**値の一致:** ✅ **完全一致**

- 数値の表記揺れのみ：整数値に `.0` が付与されるケースあり (`2`→`2.0`, `90`→`90.0`)
- 全実数値は同一

## 結論

1. **値の完全一致確認:** 全8ファイルの実数値はHEAD版とWORK版で一致
2. **差分の種類:**
   - メタデータヘッダー行の追加 → 問題なし
   - float表記の統一（`26.0`→`26`）→ 問題なし
   - 不要列の削除 → 改善
   - **カラム順序の変更（4ファイル）** → ⚠️ 列位置依存コードに影響の可能性あり
   - 行数増減（±2〜11行）→ 新しいデータの取得/一部行の欠落
3. **推奨アクション:** フロントエンド側で列位置依存のコードがないか確認（特にemployment_indices等4ファイルのカラム順序変更）
